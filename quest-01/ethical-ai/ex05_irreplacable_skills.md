# Building Irreplaceable Skills

| Skill | Description | Rating | Improvement Plan |
|-------| ------------|-------- |------------------|
|Problem Decomposition|	Breaking down problems logically|	3/5|Before writing any code, i would write;the input, the output and what should be ignored or handled specially and practice with very small problems (strings, loops, conditions) and always write pseudocode first|
|Systems Thinking|	Understanding how components interact|	2/5|Whenever I want to write code, I would read simple system explanations and summarize them in your own words.
|Critical Evaluation|Knowing when code is wrong or inefficient	|3/5|After writing code,I would add at least one test case that tries to break my code, then compare two solutions and explain why one is better (even if both work)|
|Debugging Mindset|	Investigating unexpected behavior|4/5|	When something breaks, I would use print() intentionally to inspect values, then avoid jumping to solutions (investigate first)|
|Conceptual Understanding|	Knowing WHY, not just HOW|4/5|  After solving any problem, I would explain it without code, revise once I get home, teach the concept to an imaginary beginner and focus on patterns|
